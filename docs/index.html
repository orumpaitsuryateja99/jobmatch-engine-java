JobMatch Engine Java

A Spring Boot and Next.js system that parses a resume, pulls jobs from sponsor boards and aggregators, scores each role against the resume, tailors a one-page LaTeX resume, and tracks saved or applied jobs in SQLite.

Built for fresh, relevant job search

The project automates the repetitive parts of job hunting while keeping the ranking logic transparent and deterministic.

Multi-source ingestion

Pulls from Greenhouse, Lever, Ashby, Workday, SmartRecruiters, The Muse, Adzuna, JSearch, Careerjet, Jooble, Remotive, RemoteOK, and discovery APIs.

ATS match scoring

Scores jobs against resume skills, keywords, title fit, tools, experience, domain signals, and formatting readiness.

Application tracker

Stores saved and applied jobs in SQLite so already-applied roles can be hidden from future match pulls.

Resume tailoring

Generates deterministic LaTeX resume variants from proven resume facts without inventing unsupported experience.

Docker support

Includes backend and frontend Dockerfiles plus Compose support for local full-stack runs.

Kafka events

Publishes tracker lifecycle events when Kafka is enabled, while keeping local and hosted demos simple when disabled.

Technology stack

Java backend, modern TypeScript frontend, SQL persistence, and deployable container packaging.

Run the working app

This GitHub Pages site is static. The full interactive app runs locally with the Spring Boot backend, SQLite tracker, and optional Kafka services.

Clone

Get the full source from GitHub.

git clone https://github.com/orumpaitsuryateja99/jobmatch-engine-java.git cd jobmatch-engine-java

Local dev

Start the backend and frontend together.

./run.sh

Docker + Kafka

Run the containerized stack with Kafka UI.

docker compose up --build

GitHub Pages hosting note

GitHub Pages cannot run Java, SQLite, Kafka, or private API-key-backed job pulls. This page is the GitHub-hosted project showcase. The full app is available from the repository and can be run locally or deployed to a backend-capable host.
